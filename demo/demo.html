Resource

Resource record list

GET /{ account}/{ resource-name}

uri parameters

resource-name: string

account: string

query parameters

slugified: string

filter: string

Record filter expression. Syntax examples :

  • property eq 'yop yop'
  • property neq true
  • property lt '18:56:44.123Z'
  • property gte '1977-04-22T18:56:44.123Z'
  • property gt 12
  • property lte 16.58965
  • property starts with 'abc'
  • property ends with 'def'
  • property contains 'cd'
  • nested.object.property eq 'toto'

Filter expressions are parsed and validated againts the schema, you won't be able to filter an integer property with a string litteral or a date property with a string that doesn't parse as a date.

Filter can be an ANDed list of expressions :

  • property gt 12 and property lt 16

Record

Record entry point.

Whenever modified (created, replaced or updated), the record is validated againt the resource's current schema.

DATE, TIME, and DATETIME data types are represented as string and validated against the followinbg formats :

  • DATE : yyyy-MM-dd (ex.: 1977-04-22)
  • TIME : HH:mm:ss.SSSZ or HH:mm:ss.SSSZ (ex.: 18:56:44.123Z or 18:56:44Z)
  • DATETIME : yyyy-MM-ddTHH:mm:ss.SSSZ or yyyy-MM-ddTHH:mm:ssZ (ex.: 1977-04-22T18:56:44.123Z or 1977-04-22T18:56:44Z)

DURATIONs are represented as integer values.

INTEGER, FLOAT, BOOLEAN and DURATION must not be represented as string litteral, they must be litteral from their respective types.
